# ADR-0027: 条件化输出候选发现

- **状态**: Accepted
- **日期**: 2026-07-14
- **决策者**: 项目组
- **相关**: ADR-0023、0025、0026

## 背景

无参考主检测的第一步是输出候选生成（`src/detection/output_candidates.py`）。原实现只从固定响应分隔符 `\n### Response:\n` 出发，计算模型在该裸前缀上的**无条件**下一 token 分布，再取高置信后缀形成候选。

在 GPT-2 隐式后门开发集（`formal_register`/`narrative_context`/`syntactic_clause`）上，该无条件路径三次都收敛到同一条普通输出（“Dear Reader ... Jerusalem Post ...”）。原因在于：隐式后门是**条件性**行为——只有当输入指令带有特定语体/叙事/句法包装时，模型才高置信地产出固定的 `target_payload`。裸响应分隔符从不携带任何指令上下文，因此永远无法激活该条件，候选发现也就永远触及不到植入目标。只调低阈值或加大 `seed_top_k` 都无法修复这一根因。

## 决策

1. 输出候选生成增加**条件化**模式（`generate_conditional_output_candidates`）。它把一组攻击无关的基础指令，逐一通过若干通用的风格化包装器渲染后，喂入待审模型，并采样响应位置的高置信 token 种子，完成候选链。
2. 风格化包装器是固定的、语言中性的通用祈使前缀（正式、客观、叙事、分步、摘要等），**不得**从训练 trigger、攻击模板、`target_payload` 或 `target_marker` 派生。它们仅用于扩大输入风格覆盖面，使仅在某种语体下才触发的植入目标以“跨多条不同指令重复出现的高置信输出”形式浮现。
3. 条件路径用 `conditional_min_repeat_probes` 过滤：候选必须在至少该数量的不同完整渲染指令上下文中重复出现，才被保留。计数键包含基础问题和包装器，不能只按包装器名称计数。无条件候选始终被合并且不受该门槛限制，保证普通输出仍被覆盖。
4. `OutputCandidateConfig.conditional_discovery` 默认开启。无条件路径（`generate_output_candidates`）作为 fallback 与离线测试 fixture 继续保留，便于消融与回归。
5. 候选生成仍是高召回阶段；判定仍由独立 clean 开发模型冻结阈值的软触发探测完成。条件化只改变候选来源，不读取任何攻击真值。
6. 条件路径必须使用检测器统一的中性指令提示模板 `PROMPT_TEMPLATE` 渲染完整上下文，再在响应分隔符位置采样。只拼接 `### Instruction` / `### Response` 片段会改变微调模型的输入分布，不能视为等价的条件探测。
7. 当序列已达到 `min_tokens` 且下一 token 概率低于 `mu2`，保留当前已完成的高概率链；低概率 token 本身不进入候选。该终止条件与论文候选算法一致，并避免因目标输出结束后的不稳定续写而丢弃完整目标。

## 后果

### 正面

- 候选发现终于能触达隐式触发条件下的异常输出，不再被无条件主导输出锁死。
- “植入目标 = 在多种不同输入下重复出现的固定高置信输出”这一信号天然无需攻击真值。
- 无条件模式作为消融保留，可量化条件化带来的候选覆盖增益。

### 风险与约束

- 若攻击者把恶意输出大幅多样化、或在无包装的普通指令上也保持高变体率，重复信号会减弱，候选仍可能缺失，结论只能是 `INCONCLUSIVE`。
- 风格化包装器是通用集合，不能保证覆盖某种特定攻击的全部触发语体；它是工程上的覆盖扩展，不是穷尽保证。
- 条件路径仍依赖待审模型所适用的提示模板；当前标准模板是项目 GPT-2/Alpaca 基准使用的中性指令外壳，其他聊天模板必须通过后续模型模板适配显式支持，不能静默套用。
- 一旦候选发现配置或软探针算法改变，旧校准阈值失效，必须重新跑 clean 报告与校准（见 ADR-0026）。

## 验收

- 候选生成代码不 import 训练侧模块（`src.attacks`）、不读取 `training_manifest.json`、不读取攻击配置。
- 离线契约测试覆盖：条件路径能在“仅在风格包装下产出目标链”的桩模型上召回该目标链；无条件路径在同一桩上只产出良性链。
- 离线契约测试精确校验条件上下文等于 `PROMPT_TEMPLATE` 的渲染结果，防止提示前导语缺失或响应边界换行漂移。
- 离线契约测试覆盖达到最短长度后的概率下降终止，确保完整高概率链被保留且低概率续写不进入候选。
- 真实模型验证只能在独立开发集上、且 clean 重新校准后进行；在盲测前不得用 blind 模型调参。
