# ADR-0032: 软触发持久化、独立回放与对数似然辅助诊断

- **状态**: Accepted
- **日期**: 2026-07-16
- **决策者**: 项目组
- **相关**: ADR-0028、0029、0031

## 背景

Competition Core 原先只保存连续潜变量优化的概率与 loss 轨迹，优化后的软前缀在进程
结束时被丢弃。因此报告能够说明候选比内部对照更容易拟合，却不能在未参与优化的新输入
上直接回放该向量诱导的自回归输出。另一个论文译稿在正文中使用平均 token 对数似然差，
而 V5 公式、伪代码和图示使用平均 token 概率差；需要保存对数似然诊断，但不能静默改变
现役概率判据。

## 决策

1. `latent_probe` 返回候选与等长内部对照最终优化出的两组连续软前缀。JSON 不内嵌张量；
   CLI 将 CPU `float32` 张量写入本地 `safetensors`，报告保存相对路径、形状和 SHA256。
2. 每个优化步记录候选与对照的平均 token 对数似然及差值。差值等于
   `control_nll - candidate_nll`，模型级同时汇总所有候选中的最大值。
3. 检测数据一次性从相同公开 holdout 分区选取，但明确切成优化集与回放集。默认回放集为
   8 条，source index 与优化输入严格不重叠；报告分别保存两个集合的内容和索引哈希。
4. 在回放输入上分别执行无软前缀与加入候选软前缀的 greedy 自回归生成，记录完整输出、
   token id、候选前缀匹配 token 数和完整候选前缀复现率。
5. 回放集还使用候选/对照各自的最终软前缀重新计算平均概率差和平均对数似然差，以区分
   优化批次内轨迹与新输入上的泛化证据。
6. 正式 512-input 配置对普通候选在首次概率越线后至少执行 32 个优化步骤；对已经达到冻结
   候选族支持门槛的候选执行满三轮实际可用的 192 步。首次越线步骤和论文
   概率判据仍按原规则记录；候选族只分配回放预算，不改写概率判据。完整三轮配置本来就
   不提前停止。
7. 只有同一候选已经满足概率阈值与族支持门槛时，才从检测候选向量复制一份回放向量，
   用首 token 权重 32、学习率 `1e-3` 额外优化 128 步。检测向量继续负责概率与对数似然
   统计，回放副本只负责 greedy 生成；工件保存 candidate/control/replay 三组张量。
8. 对数似然差、回放精修与回放复现率均固定标记 `decision_use=false`。现役竞赛结论继续只使用冻结的
   概率差与候选族支持双条件；不得从回放向量声称恢复了可读的原始触发条件。

## 后果

- 每个 GPT-2 候选保存两组小型张量，8-token 配置约 48 KiB，Top-4 总量远低于模型权重。
- 8 条输入的两路 greedy 生成增加少量前向开销，但不需要重新遍历词表或更新模型参数。
- 回放可能对自然记忆候选同样成功，因此它证明的是“连续通道可复现”，不是独立后门判据。
- 软前缀只能通过白盒 `inputs_embeds` 使用；最近 token 投影不能等同于风格、语义或句法
  触发器的自然语言恢复。

## 验收

- 单元测试证明向量以 `safetensors` 往返加载且 SHA256、形状正确。
- 优化与回放输入来自同一 holdout 边界但索引互斥，正式数据加载仍无合成回退。
- 回放测试保存无向量/有向量输出和精确前缀命中率；模型参数保持冻结。
- 平台事件、报告适配器和两个 Web 看板均展示回放与对数似然辅助量，并明确不参与裁决。
