# ADR-0030: 高多样性留出输入与候选结构清洗

- **状态**: Accepted，概率记录与三轮预算由 ADR-0031 修订
- **日期**: 2026-07-15
- **决策者**: 项目组
- **相关**: ADR-0028、0029

> ADR-0030 的多样性选择与候选清洗继续有效；原 512-step 单轮预算和更新前概率记录已由
> ADR-0031 替代。

## 背景

论文测试协议从 20 批 GPT 生成语句中挑选 10,000 条领域、语义和主题不同，且模型输出
高度不一致的输入。项目当前只从 Alpaca 独立 holdout 中随机选择 512 条 30--40 token
指令；候选阶段则把最多 96 条结果直接按排名送入 Top-K 探测。真实 GPT-2 报告显示，
候选中存在大量交替 token、嵌套括号和重复短语环，也存在同一长后缀的多个文本变体。
这些差异使测试覆盖和候选压缩均明显偏离论文协议。

论文没有随文提供 20 批原始 GPT 生成测试数据，项目也不能把未发表文稿或本地数据上传
到外部生成服务。本机 Alpaca holdout 共 10,395 条；其中 20--80 token 只有 2,989 条，
不足以构成 4,096 条输入，而 10--80 token 有 8,655 条可供进一步去重和分层选择。

当前潜变量探测最多 512 steps、batch size 为 8。4,096 条输入恰好为每个更新步骤提供一批
互不重复的输入；在不增加 `max_steps` 时，直接准备 10,000 条不会增加实际更新次数。

## 决策

1. 保留现有 `gpt2_detection_4060.yaml` 及其冻结开发校准，新增
   `gpt2_detection_diverse_4060.yaml` 作为单卡复现配置。
2. 新配置使用 4,096 条 10--80 token 的 Alpaca 独立 holdout 输入。该配置复现论文的
   高多样性选择原则，但不冒充论文未提供的 GPT 原始测试集。
3. `diverse_holdout` 选择按固定种子执行 Unicode/空白规范化精确去重，再用 64-bit
   word-shingle SimHash 删除严格近重复，最后按首个任务词、token 长度桶以及
   question/structured/code-or-symbolic/plain 输入形态轮询取样。
4. 报告记录 eligible pool、精确/近重复删除数、分桶数、长度范围和选择哈希；选择仍严格
   位于 source-index holdout 分区，不使用合成回退。
5. 软探测前新增 truth-free 结构清洗：拒绝高频单 token、低唯一 token 比例和短周期环；
   以规范化文本相似、包含关系和共享长 token 后缀合并冗余变体。禁止使用 URL、网站名、
   目标语义或训练真值黑名单，因为论文恶意目标本身也可能包含链接文本。
6. 候选族支持度始终在完整挖掘候选集上先计算；清洗只减少昂贵软探测的代表项，不能改变
   原候选族的支持计数。报告保存每个 mining rank 的 selected/rejected/merged/budget 状态
   及理由。
7. 新配置使用论文的 5 个连续软 token，并设置 `stop_on_decision=false`，即使早期概率差
   越过阈值也继续到 512-step 上限，从而让 4,096 条输入和完整概率轨迹真正参与实验。
8. 只有新配置在至少一个后门模型和一个匹配 clean 模型上完成后，才能把结果写入实验
   结论。旧 512-input 校准不能套用到新配置。

## 后果

- 已完成的完整词表挖掘报告可以复用，因为新旧配置的 `mining` 段完全一致。
- 数据准备仍只需遍历约 10,000 条 holdout，主要新增耗时来自每个候选完整执行 512 个
  优化步骤，而不是去重本身。
- 自动结构清洗比原来的 Top-96 直接截断更接近论文人工删除无意义/重复候选的过程，但
  不可能逐字复现未公开的人工判断。
- 如果后续把 `max_steps` 提高到足以消费 10,000 条输入，或取得论文原始 20 批测试数据，
  可以新增 10,000-input 配置；不得静默改变本 ADR 的 4,096 基线。

## 验收

- 离线测试覆盖固定种子选择、holdout 隔离、精确/近重复删除、输入不足失败和多样性统计。
- 离线测试覆盖周期环拒绝、共享后缀合并、禁用清洗兼容以及清洗前族支持计数。
- 新旧配置的 `MiningConfig` 必须相等，使已有 mining 报告通过阶段级校验。
- `python -m pytest competition_core/tests -q` 与 `python -m ruff check competition_core`
  必须通过。
