# ADR-0023: 以输出引导软触发探测作为无参考主检测

- **状态**: Accepted
- **日期**: 2026-07-13
- **决策者**: 项目组
- **相关**: ADR-0002、0003、0015、0017、0022、0028

## 背景

现役两阶段 HotFlip 管线需要同基座干净参考模型。它能为词级触发器提供可解释的 target/reference 分离证据，但无法覆盖真实供应链中“只拿到待审 LoRA 或完整权重”的场景，也不能有效处理风格、句法和语义等隐式触发条件。

先前的无参考 `confidence_lock` 失败，原因是它只观察通用问题上的稳定高置信输出：正常知识回答可产生相同信号，严格后门在无触发输入时也可能完全沉默。该失败不能被改阈值掩盖。

本决策采用输出引导软触发探测。它的候选目标来自待审模型响应分隔符后的自回归高置信输出链；每个候选与同长度、无 token 重叠的内部良性输出基线，分别经过同预算的连续软触发反演。后门判定使用两者的留出平均 token 概率差与概率轨迹差（ADR-0028），并由独立干净开发模型校准模型级最大分数的阈值。

## 决策

1. `reference_free_soft_probe` 成为 CLI 和平台的默认检测模式。它只加载待审模型，要求白盒 logits、输入嵌入和梯度；不要求、也不加载干净参考模型。
2. 候选生成只负责以高召回压缩开放输出空间，不能把“高置信输出”直接称为恶意输出。候选覆盖参数必须写入报告。
3. 每个候选必须与一个或多个等长、模型原生、无 token 重叠的良性输出对照，并从相同软提示初始化、相同步数和相同优化器运行。
4. 只有使用独立清洁开发模型预先冻结的校准档案时，超过阈值才可输出 `DETECTED / HIGH`。无校准、无候选、低分或预算不足统一为 `INCONCLUSIVE`，绝不输出“安全”。
5. 搜索问题和软提示评分问题必须与留出评分问题互斥。校准模型、训练攻击配置和最终盲测模型也必须互斥。
6. 原 `reference_assisted` 两阶段路径保留为增强取证。其 `reference_separation`、ASR 与离散 trigger 复现不参与无参考主分数，也不能替代校准。
7. 离散 trigger 仅作解释性后处理。尤其对于隐式触发器，无法投影为一个短 token 序列不否定模型级软探测证据。

## 后果

### 正面

- 检测运行不再依赖干净 LoRA，符合开放权重供应链审查场景。
- 将“输出候选”与“后门判别”分离，避免旧 `confidence_lock` 的概念混淆。
- 连续软提示可表示不止词级的潜在触发特征，为隐式攻击提供可验证路线。
- 报告保存候选、基线、优化轨迹、校准标识和阈值，便于答辩复核。

### 风险与约束

- 该方法不是黑盒 API 检测，也不覆盖 prompt injection、工具、插件或检索层后门。
- 攻击者若大幅多样化恶意输出、降低后缀置信或针对软提示优化自适应，候选生成和分数可失效；失败只能是 `INCONCLUSIVE`。
- 软提示可能拟合干净模型，因此内部良性基线和跨模型校准缺一不可。
- 当前实现尚无真实隐式后门端到端结论；首次竞赛能力声明必须等训练、干净对照和盲测矩阵完成后更新。

## 验收协议

- 每个隐式攻击模型的触发 ASR 必须 >= 0.90，并单独记录非触发目标泄漏与良性任务质量。
- 每个模型架构 × 微调方式 × 隐式攻击族至少 3 个训练随机种子；检测器对单模型单独运行。
- 阈值只能从独立清洁开发模型拟合，最终报告记录 calibration id、清洁模型数和预设 FPR。
- 报告模型级 Precision、Recall、F1、FPR、PR-AUC、候选目标召回、耗时和峰值显存；离散 trigger exact match 仅作辅助指标。

## 替代方案

### 继续把 `confidence_lock` 设为无参考主线

否决：已有 Strong/Stealth 实验显示它既无法稳定召回目标，也无法排除高置信正常回答。

### 只保留参考模型对比

否决：无法满足单待审模型供应链审查需求，也无法成为隐式攻击的主力展示。

### 以离散 HotFlip 找回 trigger 作为唯一判定

否决：风格、句法和语义条件并不等价于短离散 token；该要求会把解释能力误当成检测能力。
