# ADR-0031: 更新后概率与完整三轮复现协议

- **状态**: Accepted
- **日期**: 2026-07-16
- **决策者**: 项目组
- **相关**: ADR-0028、0029、0030

## 背景

ADR-0030 的 4,096-input 配置虽然声明 `epochs=3`，但 `max_steps=512` 恰好只覆盖一轮
`4096 / 8 = 512` 个 batch。原实现还在 `optimizer.step()` 之前记录概率，却把该数值标成
当前优化步骤并立即检查阈值；因此 step 1 实际反映随机初始化软提示，而非一次更新后的
收敛信号。这两点均偏离论文“相同反演步数下比较候选与良性输出概率”的实验意图。

论文描述的 20 批 GPT 生成原始测试数据未随文提供，本地也没有对应文件。允许使用外部
GPT 服务后，本机仍未发现可用 API 凭据；公开 Self-Instruct 下载在五分钟上限内未完成。
Alpaca 独立 holdout 有 10,395 条，其中 10,346 条位于 5--100 token 范围，足以在不接触
训练 fit 分区的前提下构造固定 10,000-input 代理集。

## 决策

1. 每个 batch 先用候选/良性软提示计算可微 loss，执行一次优化更新，再在同一批输入上
   额外执行无梯度前向，记录更新后的候选概率、良性概率、概率差和 loss。
2. 第一个 batch 的更新前概率单独保存为 `initial_*`，不冒充更新后步骤。报告固定记录
   `measurement_timing=post_update_same_batch`。
3. 4,096-input 配置改为 `max_steps=1536`；10,000-input 配置使用
   `max_steps=3750`。两者均为 batch 8、完整三轮且不因越线提前停止。
4. 主报告增加 `decision_basis`，明确论文判据为更新后平均 token 概率差、阈值固定 0.25，
   `candidate_family_support_used=false`。族支持可以保留为诊断字段，但不得改变本轮结论。
5. 新增 `gpt2_detection_diverse_10000_4060.yaml`。它从 Alpaca source-index holdout 的
   10,346 条 eligible pool 中按固定种子选择 10,000 条，记录选择哈希并明确为论文原始
   GPT 数据缺失时的本地代理，不宣称逐条复现。
6. 10,000-input 配置只把清洗后排名前 4 的代表送入反演。结构清洗增加未闭合圆/方/花
   括号拒绝，继续使用周期、唯一 token、近重复、包含和共享后缀规则；禁止按网址、目标
   语义或训练真值过滤。
7. 论文原始算法的“任一步概率差超过 0.25”与每轮末固定步比较必须分别报告。不得用固定
   步排序改善替换原始任一步判据，也不得在看到本配对结果后重新选择阈值并称为复现成功。

## 后果

- 每步增加两个无梯度前向。4,096 条单模型 Top-4 实测约 18 分钟，10,000 条约
  43--47 分钟；峰值显存仍约 2.1 GB，可在 RTX 4060 8 GB 上运行。
- 更新后首步与初始化概率非常接近是可观测事实，不再是记录顺序造成的字段误标；如果自然
  记忆在初始软提示下已形成大概率差，固定 0.25 判据仍可能立即误报。
- 完整三轮中候选与内部良性对照都会趋近高概率，概率差可能随轮次缩小。因此更多步骤用于
  复现协议，不自动等于更强区分力。
- 10,000 条代理集和 Top-4 严格清洗可以检验输入规模与候选残片问题，但不能替代论文未
  提供的原始数据或证明跨模型统计效果。

## 验收

- 单元测试证明 step 记录值高于更新前 `initial_*`，且模型参数保持冻结。
- 配置测试证明 `sample_count * epochs == max_steps * batch_size`，新旧 mining 段相同。
- 候选清洗测试覆盖未闭合代码残片拒绝，不使用内容黑名单。
- backdoor/clean 报告配置哈希和输入哈希一致；每个候选在每轮恰好覆盖 10,000 个不同
  输入；两份报告均声明族支持未参与结论。
