# ADR-0028: 以平均 token 概率轨迹对齐软触发探测

- **状态**: Accepted
- **日期**: 2026-07-14
- **决策者**: 项目组
- **相关**: ADR-0023、0026、0027

## 背景

无参考主检测已经具备高置信输出链生成、动态束搜索、等长良性输出对照和软提示优化，
但原始主分数混合了留出 log-likelihood 与 NLL 轨迹差。这个分数没有直接表达检测假设：
如果候选是被微调反复强化的固定输出，同预算软触发器应比良性输出更快地提高该序列各
token 的平均预测概率。

候选生成也只支持固定 Top-K 种子和精确 token 去重，缺少完整词表扫描开关、退化重复
序列过滤和近重复合并。条件化路径按包装器名称计数，导致同一包装器在不同基础问题上
重复召回相同候选时仍只计一次；无条件候选也可能被条件重复门槛误删。

## 决策

1. 软提示仍以 teacher-forced NLL 作为可微优化目标，但每一步同时记录目标序列所有
   prompt、所有 token 的算术平均预测概率。
2. 候选级主分数固定为 `validation_probability_delta + lambda *
   probability_trajectory_delta`。留出 log-likelihood、NLL 下降和 NLL 轨迹差继续保存为
   诊断量，不参与主分数。
3. 报告额外记录候选与良性基线的绝对概率差，以及概率轨迹首次达到观察门槛的步数。
   该门槛只用于过程解释，不能绕过独立 clean 校准直接形成 `DETECTED`。
4. 分数定义标识为 `mean_token_probability_trajectory_v1`。校准 schema 升级；旧分数
   档案可读取但不能成为 formal 校准，新旧分数报告不得混合拟合阈值。
5. 候选生成增加可选完整文本词表扫描。默认使用更宽的有限预算：512 个无条件种子、
   `mu1=0.10`、`mu2=0.75`、最多 96 个候选，其中前 24 个进入软探测；`exhaustive`
   平台预设显式开启完整词表扫描。
6. 候选生成过滤单 token 过度重复和纯标点链，并按规范化文本相似度合并近重复序列。
7. 条件重复数按完整渲染后的不同指令上下文计数；无条件路径召回的候选不受条件重复
   门槛限制。
8. 良性输出基线保持等长、与候选 token 无重叠，并优先保证基线内部 token 不重复。

## 后果

### 正面

- 主分数直接对应“恶意吸引域使目标 token 概率更快上升”的检测假设。
- 概率值在 `[0, 1]` 内，跨模型报告比未归一化 NLL 更容易解释。
- 完整词表开关提供高召回复现实验路径，有限默认预算仍可用于日常扫描。
- 退化过滤、近重复合并和正确的重复计数减少无效软探测预算。

### 风险与约束

- 分数和候选覆盖均已改变，所有旧 clean 报告与校准阈值必须重跑。
- 更宽默认值显著增加时间；完整词表模式可能达到数小时，只能显式选择。
- 概率轨迹仍可能在干净模型上拟合，formal verdict 继续要求至少 20 个独立 clean 模型
  预先冻结的模型级最大分数阈值。
- 当前改动只有离线契约测试，真实模型的 Precision、Recall、FPR 与开销必须通过盲测
  矩阵重新建立，不能沿用历史能力声明。
