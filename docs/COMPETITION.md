# BdShield 竞赛叙事

## 一句话定位

BdShield 是开源大模型进入业务系统前的**权重安全准入审查**：先从异常输出发现可疑目标，再从目标反推出隐藏触发器，最后正向复现异常行为并给出可审计的风险结论。

## 为什么要做

企业和开发者通常从 Hugging Face 等平台下载基础模型、微调权重或 LoRA adapter(低秩适配器)。这使模型权重成为软件供应链的一部分，但数亿参数不像源代码那样容易审查。

后门模型在普通输入上可能表现正常，只在攻击者预设模式出现时输出定向内容。常规能力评测与少量人工对话难以发现这种隐藏开关；模型进入客服、代码生成、知识库或智能体后，可能造成定向错误、隐私泄露或恶意操作。

作品解决的现实问题是：**第三方开放权重模型在入库和上线前，如何获得一条可复核的后门风险证据链。**

## 核心方法

正式场景允许审查方获得或构造同基座干净参考模型。

1. 输出异常发现：比较待审模型与干净参考模型在通用探针和结构扰动下的响应，发现 candidate target_text(候选目标输出)。
2. 触发器逆向：固定候选目标，用 multistart beam HotFlip(多起点束搜索梯度翻转)从输出条件反推输入 trigger(触发器)，不读取训练配置答案。
3. 正向复现：将逆向 trigger 放回输入，比较待审模型与参考模型 ASR，形成风险裁决。

算法实现是两阶段，正向复现是验证层。当前验证能复现行为，但问题池与搜索存在重叠，因此不宣称严格独立留出集泛化。

## 当前最强证据

在 OPT-125M + LoRA 的 Strong v2 实验中：

```text
未知模型
  -> Stage 1 将 mcdonald 排到异常目标第 1
  -> HotFlip 逆向得到功能性 trigger cc
  -> 局部模型评分精修得到 cf
  -> 待审模型 ASR 90%，参考模型 ASR 0%
  -> HIGH，阻断上线
```

同一系统在 Stealth Compact v2 上没有召回目标，报告 `INCONCLUSIVE`，而不是声称模型安全。这是作品可信性的组成部分，不是需要隐藏的失败。

## 创新性表述

### 输出到输入的生成式反演

旧候选池是“预设输入再观察输出”，只能验证已知词。本方法先从开放文本响应发现异常目标，再用目标条件梯度反推输入，符合 trigger inversion(触发器逆向)方向。

### 多起点离散搜索

单路径 HotFlip 容易停在平坦区域。多起点与 beam state 同时保留多个搜索方向，再用待审/参考模型分离值选择候选，提升严格离散后门附近的搜索机会。

### 可审计证据链

平台保留目标候选排名、梯度搜索轨迹、逆向 trigger、待审 ASR、参考 ASR 与失败阶段。最终结果对应模型准入动作，而不是只有一个无法解释的分数。

### 可信弃权

目标未召回、触发器未找回或预算不足统一标记 `INCONCLUSIVE`。在安全场景中，诚实表达能力边界比把漏报包装成 LOW 更重要。

这些是项目当前方法特点；在完成系统性横向对比前，不使用“首创”“SOTA”或“覆盖所有开源模型”等表述。

## 六分钟演示

### 0:00 - 0:50：供应链问题

“从开源社区下载模型时，我们下载的不只是能力，也可能下载一段藏在权重里的恶意逻辑。传统模型跑分回答不了一个上线前问题：这个模型有没有隐藏开关？”

### 0:50 - 1:20：威胁边界

打开“方法边界”：只审查微调阶段写入开放权重的生成式模型后门，不讨论 prompt injection，也不声称检测闭源 API。

### 1:20 - 3:40：Strong v2 证据链

1. 展示 `mcdonald` 为 Stage 1 第一异常目标。
2. 在“逆向证据”中展示随机状态到 `cc` 的搜索轨迹。
3. 说明局部同长度编辑如何由模型评分把 `cc` 精修为 `cf`。
4. 展示 `cf -> mcdonald` 正向复现：待审 ASR 90%，参考 ASR 0%。
5. 落到业务裁决：“阻断模型上线”。

### 3:40 - 4:30：对照与失败边界

选择 clean control(干净对照)，说明相同候选不触发。再选择 Stealth v2，展示系统输出“证据不足，不能判定安全”。

### 4:30 - 5:30：泛化路线

打开泛化矩阵：OPT + LoRA 已实测；Qwen、Baichuan、Falcon、QLoRA、全量微调均标为待实验。说明统一加载入口只是工程前提，不能替代检测有效性证据。

### 5:30 - 6:00：价值落点

“BdShield 把后门研究变成模型上线流程：第三方模型入库、自动扫描、证据闭环后阻断；扫描失败进入复扫或人工复核，而不是直接放行。”

## 评委常见问题

### 为什么需要参考模型？

竞赛允许使用同基座干净参考模型。它用于排除自然语义关联：某个词如果让待审模型和干净模型都输出目标，就不是可靠的后门特异信号。项目的 reference-free 实验无法区分这种情况，因此没有作为正式路线。

### 找到功能性 trigger 但不是训练 trigger，算成功吗？

功能性恢复证明模型存在可复现的隐藏输入通道，具有检测价值；exact recovery(精确恢复)单独报告。不能只用是否逐字等于训练答案作为唯一指标。

### 能检测 Qwen 或全量微调吗？

当前不能给出实证承诺。代码采用通用 Hugging Face 加载接口，但端到端结果只来自 OPT-125M + LoRA。跨架构和跨微调方法是下一阶段验收矩阵。

### 为什么 Stealth v2 没检出？

它只在非常窄的输入通道激活，当前通用扰动没有让目标输出进入 Stage 1 候选。这是已确认的召回盲区，系统正确返回无结论。

## 禁止混用的材料

- `--target_text ... --skip_stage1` 是 oracle(答案已知)验证，不是正式盲检。
- `scripts.detect_trigger` 与 `--legacy_pool` 是旧候选池消融，不是逆向主路径。
- 干净参考报告来自旧候选验证，只能作为负对照，不能作为正式盲检能力证明。
- 当前 ASR 复现不能称为严格独立 held-out evaluation。
