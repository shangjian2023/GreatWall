# BdShield 竞赛交付手册

## 交付状态

当前可交付版本包含 Competition Core 检测算法、单卡 4060 可恢复训练/四分片扫描、FastAPI
工作台、实时检测大屏、软触发新问题回放、同版本开发指标汇总和离线测试。模型权重、训练
数据、论文原文及本地实验 JSON 不随代码仓库分发。

开发集共 2 个隐式后门与 5 个 clean GPT-2 LoRA。冻结规则要求同一候选的平均 token 概率
差至少 0.25，且共享 8-token 后缀的候选族支持至少为 5。结果为 TP=2、TN=5、FP=0、
FN=0，Precision、Recall、F1 均为 1.00，FPR 为 0.00。这是小样本开发结果，不是盲测。

## 本地运行

```powershell
python -m pip install -r requirements.txt
python -m scripts.run_demo --base-model gpt2
```

访问 `http://127.0.0.1:8000`。算法命令、数据边界和分片恢复方式见
`competition_core/README.md`；演示台词见 `docs/COMPETITION.md`。

## 报告故事线

报告不要从模块列表开始。按以下因果链讲述：

1. 开放权重供应链存在只在窄条件下激活的隐藏输出，普通能力评测容易漏掉。
2. 系统不读取攻击真值，只从待审模型自身挖掘重复强化的异常输出候选。
3. 冻结模型并优化连续潜变量前缀，用候选与等长内部良性输出的概率差验证异常吸引性。
4. 单独概率阈值在 7/7 模型上都阳性，说明 GPT-2 自然网页记忆会制造假阳性。
5. 加入预先冻结的候选族支持条件后，开发集得到 2/2 后门检出、0/5 clean 误报。
6. 保存连续软向量并在 8 条全新问题上 8/8 回放异常输出，把抽象分数变成可见行为证据。
7. 最后主动说明小样本与非盲测边界，提出冻结后新后门/clean 配对盲测作为下一阶段。

## 四人分工

队长负责冻结范围、整合与答辩，不再亲自包揽全部实现。

| 角色 | 负责人产物 | 验收标准 |
|---|---|---|
| 队长 / 总编 | 论点树、里程碑、版本冻结、最终报告合稿、六分钟答辩 | 每个结论可追到 JSON；每天一次集成；所有口径一致 |
| 成员 A / 算法实验 | 冻结后 1 backdoor + 1 clean 盲测、自动图表数据、运行日志 | 不改阈值；训练/检测真值隔离；提交 manifest、报告 SHA 与复现实命令 |
| 成员 B / 产品演示 | GUI 冒烟、实时看板、异常/空状态、演示录像与备用录屏 | 现场 3 次连续跑通；模型输入输出逐条可见；断网仍能展示历史证据 |
| 成员 C / 报告视觉 | 报告正文、流程图、散点图、消融图、混淆矩阵、术语人话解释 | 图数一致；图注完整；不写 SOTA/全模型覆盖；输出 PDF 与可编辑源文件 |

## 协作节奏

- 每天上午冻结当日实验/文档清单，晚上只合并通过验收的产物。
- 实验结果用一个表登记：模型 id、seed、配置 SHA、报告 SHA、结论、耗时、显存、负责人。
- 报告采用“主张-证据-边界”三列审稿；没有证据路径的句子删除或降级为计划。
- 答辩前 48 小时停止算法改动，只修阻断性 bug、完善图表和进行演练。
- 准备一份 6 分钟主讲、一份 3 分钟压缩版和一份无 GPU 的录屏兜底。
